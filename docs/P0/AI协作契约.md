《AI 协作契约》

团队名称：辛巴巴把鹿比啦
日期：2026.04.05

第一条：AI 使用原则
- AI 是工具，工程师是责任主体
- 所有 AI 输出必须经过人工审查后才能合入项目

第二条：AI 使用范围
- 允许使用 AI 的场景：文档撰写、代码补全、单元测试生成
- 禁止直接使用 AI 输出（需额外审查）的场景：代码逻辑设计、核心算法实现
- 完全禁止使用 AI 的场景：核心模块开发、性能优化、code review

第三条：过程记录要求
- 所有与 AI 的交互 Prompt 必须保留日志
- Git commit message 必须标注 [AI-assisted] 或 [Human-written]
- 每阶段填写《AI 协作反思日志》

第四条：代码合入规则
- AI 生成的代码必须通过至少一名团队成员的 Code Review
- 每位成员每个编码阶段至少有 2 个完全手写的核心函数

第五条：违约处理
- 团队内部约定的违约处理方式：罚款+公开道歉
