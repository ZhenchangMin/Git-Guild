# Git Guild P4 Sprint 合并任务看板

## 1. 看板说明

本文档主要用于记录 Git Guild 项目 P4 编码开发阶段的 Sprint 合并任务安排，覆盖任务拆解、优先级、负责人、预计工时、状态、依赖关系和完成标准。

## 2. 状态、优先级和负责人

### 2.1 状态定义

| 状态          | 含义                |
| ----------- | ----------------- |
| Todo        | 尚未开始              |
| In Progress | 正在进行              |
| Blocked     | 被依赖或技术问题阻塞        |
| Review      | 已完成初稿，等待组内检查或联调确认 |
| Done        | 已完成并通过验收          |

### 2.2 优先级定义

| 优先级  | 含义                |
| ---- | ----------------- |
| P0   | P4 必须完成，影响核心演示链路  |
| P1   | 核心链路完成后完成，增强演示完整度 |
| P2   | 可选增强，不影响 P4 通过    |
| Test | 测试、CI、质量保障        |
| Doc  | 文档、实验记录、演示材料      |
