## 一、说明

本日志用于记录 Phase 4 编码与联调阶段发现并修复的 Bug，每条记录覆盖四要素：

1. **问题现象** — 用户或测试观察到的**可复现行为**
2. **根因分析** — 为什么会发生（直接原因 + 深层原因）
3. **修复方案** — 改动了什么、影响范围、是否有备选方案
4. **验证结果** — 如何确认修复且未引入回归

### 记录规范

- 每个 Bug 一个条目，编号 `BUG-001`、`BUG-002` …，按发现时间升序
- 关联 Git 提交、PR、Issue 时使用短哈希或编号（如 `#42`、`a1b2c3d`）
- 阻塞演示或主流程的 Bug 在标题前加 `[阻塞]` 前缀
- 进入 "AI 调试对决" 实验的 Bug 同时在该日志中登记

### 字段取值

- **严重级别**：阻塞 / 高 / 中 / 低
- **状态**：待修复 / 修复中 / 已修复 / 已验证 / 不修复（需说明原因）
- **模块**：用户管理 / 需求发布 / 任务 / 评价 / 消息通知 / 仓库托管 / 前端 / 后端 / 数据库 / CI / 其它
