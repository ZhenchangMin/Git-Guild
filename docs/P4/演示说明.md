# Git Guild P4 演示说明

> 最后更新：2026-06-07  

## 1. 演示目标

本次演示使用本地可运行版本展示 Git Guild 的 P4 核心闭环：

```text
维护者发布悬赏任务 -> 管理员审核上架 -> 冒险家接取任务
-> 工作台克隆并推送代码 -> 提交柜台自动创建 PR
-> 维护者审核通过 -> PR 自动合并 -> XP 与成长记录更新
```

| --- | --- | --- |
| 前端 | `http://localhost:5173` | Vue 3 + Vite，本地开发服务器 |

确认 `gitguild-mysql`、`gitguild-redis`、`gitguild-gitea` 均为 `healthy` 后，再启动后端和前端。

## 6. 核心演示路径

### 6.1 维护者发布任务

1. 打开 `http://localhost:5173`。
2. 使用 `guild` 登录。
3. 进入发布悬赏任务入口。
4. 使用预置仓库和 Issue：

```text
仓库：hello-world-mvp
仓库地址：http://localhost:3000/spike-admin/hello-world-mvp
Issue：#1 Add hello world file
```

5. 填写任务信息并提交。

### 6.2 管理员审核上架

1. 使用 `admin` 登录。
2. 打开管理员审核入口。
3. 找到刚发布的任务。
4. 选择审核通过。

### 6.3 冒险家接取任务

1. 使用 `advent` 登录。
2. 进入 Quest Board / 悬赏任务板。
3. 打开刚发布的任务详情。
4. 点击接取任务。

### 6.4 工作台克隆并推送代码

1. 进入冒险家工作台。
2. 使用 Clone & Push 卡片中的克隆地址。
3. 按页面提示克隆仓库、切换任务分支、修改文件、提交并推送。

说明：演示用克隆地址会携带本地 Gitea 凭据或 token，用于避免 `git push` 时再输入密码。该方式只适用于本机演示环境。

### 6.5 提交柜台自动创建 PR

1. 冒险家进入提交柜台。
2. 选择当前任务。
3. 填写成果说明和自检清单。
4. 提交成果。

### 6.6 维护者审核成果

1. 使用 `guild` 登录。
2. 进入维护者审核队列。
3. 打开冒险家的成果提交。
4. 点击审核通过。

### 6.7 成长档案与排行榜

1. 使用 `advent` 登录。
2. 打开个人档案或成长信息页面。
3. 查看 XP、等级、完成任务数和贡献记录。
4. 打开排行榜查看排名变化。
