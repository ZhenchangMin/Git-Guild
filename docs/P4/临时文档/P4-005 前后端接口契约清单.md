# P4-005 前后端接口契约清单

## 1. 文档目的

本文档用于 P4 编码阶段统一前端页面、后端 API、字段、状态和错误提示。后续前端接入、后端开发、Gitea 适配、联调测试均以本清单为初始契约。

本清单遵循以下已有设计：

- 平台采用自有账号体系。
- 代码托管底座集成自建 Gitea。
- GitHub 到平台/Gitea 采用单向同步。
- MVP 优先完成仓库导入、Issue 浏览、任务发布、任务接取、成果提交、审核、轻量成长。
- 推荐匹配、排行榜、徽章纳入 P4 范围，但优先级为 P2，不阻塞主链路。

## 2. 通用约定

| 项      | 约定                                                    |
| ------ | ----------------------------------------------------- |
| API 前缀 | `/api/v1`                                             |
| 鉴权     | 登录后使用 `Authorization: Bearer <token>`                 |
| 响应格式   | `{ "code": "OK", "message": "success", "data": ... }` |
| 分页字段   | `page`、`size`、`total`、`items`                         |
| 时间格式   | ISO 8601 字符串，如 `yyyy-mm-ddThh:mm:ss+08:00`            |
| ID 类型  | 后端主键用 Long/String 均可，但前后端统一以字符串展示                     |
| 文件上传   | MVP 只支持上传/提交变更，不做网页编辑器                                |

## 3. 核心状态枚举

| 类型       | 状态                                                                                                                            | 说明                                         |
| -------- | ----------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------ |
| 用户角色     | `VISITOR`、`ADVENTURER`、`MAINTAINER`、`ADMIN`                                                                                   | 访客、冒险家、委托人、管理员                             |
| 仓库同步     | `SYNCED`、`SYNCING`、`FAILED`、`PENDING`                                                                                         | 已同步、同步中、同步失败、待同步                           |
| Issue 状态 | `OPEN`、`CLOSED`                                                                                                               | 由 Gitea/GitHub 同步                          |
| 悬赏任务状态   | `DRAFT`、`PENDING_ADMIN_REVIEW`、`OPEN`、`ASSIGNED`、`SUBMITTED`、`IN_REVIEW`、`CHANGES_REQUESTED`、`COMPLETED`、`REJECTED`、`OFFLINE` | 草稿、待管理员审核、可接取、已接取、已提交、审核中、退回修改、已完成、已驳回、已下架 |
| PR 状态    | `NOT_CREATED`、`OPEN`、`CHECKING`、`REVIEW_REQUESTED`、`CHANGES_REQUESTED`、`MERGED`、`CLOSED`                                      | 未创建、打开、检查中、待审核、退回修改、已合并、已关闭                |
| 成果提交状态   | `DRAFT`、`SUBMITTED`、`IN_REVIEW`、`APPROVED`、`CHANGES_REQUESTED`、`REJECTED`                                                     | 草稿、已提交、审核中、通过、退回修改、驳回                      |
| 管理员审核状态  | `PENDING`、`APPROVED`、`RETURNED`、`FLAGGED`、`OFFLINE`                                                                           | 待处理、通过上架、退回补充、标记异常、下架                      |

说明：`VISITOR` 为未登录访问态，主要用于前端路由、页面展示和权限提示；后端需要鉴权的写操作接口不接受访客操作。访客可浏览公开任务和公开仓库信息，但不能接取任务、提交成果、发布任务、审核成果或执行管理员操作。

## 4. 页面-接口-字段表

### 4.1 登录入口页

| 页面动作 | API | 方法 | 请求字段 | 响应字段 | 错误提示 |
| --- | --- | --- | --- | --- | --- |
| 注册账号 | `/auth/register` | POST | `username`、`password`、`displayName`、`role` | `userId`、`username`、`displayName`、`role` | 用户名已存在、密码不符合规则 |
| 登录 | `/auth/login` | POST | `username`、`password` | `token`、`user` | 账号或密码错误 |
| 获取当前用户 | `/auth/me` | GET | 无 | `userId`、`displayName`、`role`、`xp`、`level` | 登录状态失效 |
| 访客进入 | 前端本地状态 | - | 无 | `role=VISITOR` | 访客只读，不允许接取/提交/审核 |

访客权限约束：

- 允许访问：登录入口、悬赏任务板、任务详情、公开仓库/Issue 摘要、前台向导。
- 禁止访问：接取任务、创建分支、上传 commit、发起 PR、提交审核、发布任务、审核成果、管理员上架审核。
- 前端遇到访客点击受限操作时，应提示“请先登录后再执行该操作”。

## 5. 通用错误码与前端提示

| 错误码                            | HTTP | 前端提示                  |
| ------------------------------ | ---- | --------------------- |
| `UNAUTHORIZED`                 | 401  | 登录状态已失效，请重新登录         |
| `FORBIDDEN`                    | 403  | 当前账号没有执行该操作的权限        |
| `VALIDATION_ERROR`             | 400  | 表单信息不完整，请检查必填项        |
| `REPOSITORY_IMPORT_FAILED`     | 502  | 仓库导入失败，请检查地址、权限或稍后重试  |

| `QUEST_NOT_FOUND`              | 404  | 任务不存在或已下架             |
| `QUEST_NOT_ACCEPTABLE`         | 409  | 当前任务状态不可接取            |
| `QUEST_ALREADY_ASSIGNED`       | 409  | 任务已被其他用户接取            |

| `SUBMISSION_NOT_FOUND`         | 404  | 提交记录不存在               |
| `SUBMISSION_NOT_REVIEWABLE`    | 409  | 当前提交状态不可审核            |
| `QUEST_DRAFT_NOT_FOUND`        | 404  | 任务草稿不存在               |
| `QUEST_DRAFT_NOT_EDITABLE`     | 409  | 任务草稿当前不可编辑            |
| `REVIEW_APPLICATION_NOT_FOUND` | 404  | 任务上架申请不存在             |
| `NOTIFICATION_NOT_FOUND`       | 404  | 通知不存在                 |
| `EXCEPTION_NOT_RETRYABLE`      | 409  | 当前异常不可重试              |
| `ADMIN_REVIEW_REQUIRED`        | 409  | 任务需要管理员审核通过后才能上架      |

## 6. 联调优先级

| 优先级 | 接口范围 | 原因 |
| --- | --- | --- |
| 第一批 | 登录、当前用户、任务列表、任务详情、接取任务 | 前端悬赏任务板和任务详情必须先跑通 |
| 第二批 | 仓库列表、仓库导入、同步状态、Issue 列表 | 委托人发布任务和工作台依赖仓库上下文 |
| 第三批 | 分支、commit、PR、成果提交 | 冒险家主链路依赖 |
| 第四批 | 委托人审核、管理员上架审核、异常处理 | 演示闭环和课堂管理员操作依赖 |
| 第五批 | 成长记录、通知、反馈归档 | 轻量成长和持续参与反馈 |
| 第六批 | 推荐、排行榜、徽章 | P2 增强，不阻塞 P0/P1 |
