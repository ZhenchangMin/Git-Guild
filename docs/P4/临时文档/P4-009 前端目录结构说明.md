# P4-009 前端目录结构说明

## 1. 文档目的

本文档记录 P4-009 “拆分前端结构”完成后的前端目录、路由、页面、数据、API 占位和后续扩展方式。后续前端接入接口、增删页面、继续拆分工作台时，应优先参考本文档。

当前前端改动已推送到 `front` 分支，提交为：

- `4078f75 feat(frontend): split prototype into routed pages`

## 2. 当前结构概览

```text
frontend/
  package.json
  package-lock.json
  src/
    App.vue
    main.js
    style.css
    assets/
    api/
    components/
    config/
    data/
    pages/
    router/
    stores/
```

核心变化：

- `App.vue` 已简化为全局路由出口，只保留 `<RouterView />`。
- `main.js` 已挂载 `vue-router`。
- 原集中式大原型壳已移除，不再使用 `screen`、`activeRoom`、`window.location.hash` 作为主页面路由机制。
- 各主要页面已拆分到 `src/pages/` 下。
- API 层、store 层和导航配置已建立占位，便于后续接后端接口。

## 9. 新增页面规范

新增页面建议按以下步骤：

1. 在 `src/pages/<domain>/` 下新增页面组件。
2. 在 `src/router/routes.js` 注册路由。
3. 如果需要导航入口，在 `src/config/navigation.js` 中补充对应角色菜单。
4. 如果需要后端数据，优先在 `src/api/` 中新增或复用 API 方法。
5. 页面独有临时数据可以先放组件内；跨页面数据再放 store。

不要再把新页面塞回单个大组件，也不要用 `window.location.hash` 模拟页面跳转。
