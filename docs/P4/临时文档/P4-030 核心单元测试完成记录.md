## 4. 主要新增测试点

### 4.1 管理员审核服务

新增 `AdminQuestServiceImplTest`，覆盖管理员审核流程中的关键状态转换和异常路径：

- 查询待审核任务时，将非法分页参数修正为安全范围。
- 管理员审核通过后，任务状态从 `PENDING_ADMIN_REVIEW` 变为 `PUBLISHED`。
- 管理员退回任务后，任务状态变为 `REJECTED`，并保存退回原因。
- 非管理员调用审核接口时返回 `FORBIDDEN`，不会继续读取任务或保存审核记录。
- 对非待审核状态任务执行上架/退回时返回 `QUEST_NOT_REVIEWABLE`。
- 执行下架时，任务状态变为 `CLOSED`，并取消所有 `ACTIVE` 接取记录。
- 对已经关闭的任务重复下架时返回 `QUEST_ALREADY_CLOSED`。

### 4.2 任务发布、提交审核和接取

在 `QuestServiceImplTest` 中补充以下测试：

- 管理员不能作为普通冒险家接取任务。
- 非 `PUBLISHED` / `IN_PROGRESS` 状态的任务不能被接取。
- 发布者可以将草稿任务提交到管理员审核，任务进入 `PENDING_ADMIN_REVIEW`。
- 非发布者不能提交任务审核，返回 `FORBIDDEN`。

这些测试主要覆盖任务状态流转和权限边界。

### 4.3 成果提交

在 `SubmissionServiceImplTest` 中补充以下测试：

- 管理员不能提交普通任务成果。
- 非 `IN_PROGRESS` / `IN_REVIEW` 状态的任务不能提交成果。
- PR 不属于任务所在仓库时返回 `PR_NOT_FOUND`，避免错误关联其他仓库的 PR。

这些测试用于保证任务、接取记录、PR 三者之间的关系不会被错误绕过。

### 4.4 维护者审核与成长激励

在 `ReviewServiceImplTest` 中补充审核结果对成长模块的调用断言：

- 审核通过时调用 `GrowthService.grantQuestCompletion(...)`，发放 XP、等级和贡献记录。
- 审核要求修改时不会调用成长发放逻辑。

这保证了 P4-020 审核反馈与 P4-022 成长激励之间的联动被单元测试保护。

### 4.5 Controller 入口测试

新增和补强 Controller 测试：

- `QuestControllerTest`
  - 任务发布接口返回 `201 Created`。
  - 任务大厅查询接口正确解析筛选、排序、分页参数。
  - 任务接取接口返回 `201 Created`。
  - 通过 `ArgumentCaptor` 验证当前用户 ID 和请求参数正确传给 Service。

- `AdminQuestControllerTest`
  - 管理员待审核任务列表接口返回分页数据。
  - 管理员审核接口返回 `201 Created`。
  - 通过 `ArgumentCaptor` 验证管理员 ID、审核决策、审核原因和可见性参数正确传给 Service。

- `SubmissionControllerTest`
  - 补充对成果提交请求体的捕获断言。
  - 补充对审核请求体的捕获断言。
  - 确认当前用户 ID 正确传递给业务服务。
