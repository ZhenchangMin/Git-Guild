# P4-003 前端页面清点

## 1. 清点范围

本次清点对象为 `frontend/src` 下现有 Vue 前端原型，重点确认 P4 看板中要求的登录、悬赏任务板、任务详情、工作台、提交柜台、管理员审核、成长页是否已经存在，以及后续应保留、调整或删除的内容。

## 2. 总体结论

现有前端原型已经覆盖 Git Guild MVP 的主要演示链路，可以作为 P4 前端开发起点继续推进。当前主要问题不是缺页面，而是页面职责集中、mock 数据较多、部分文案存在编码异常、真实接口尚未接入。

P4 阶段建议保留现有视觉原型和页面结构，优先完成以下调整：

- 将 `App.vue` 中过重的页面逻辑逐步拆分到独立组件。
- 将 mock 数据替换为后端接口或统一 mock service。
- 修正文案编码异常，保证课堂演示时中文可读。
- 按 MVP 收敛功能，优先保证任务发布、任务浏览、任务接取、成果提交、审核和轻量成长闭环。

## 4. 需要优先修正的问题

| 问题 | 影响 | 建议处理顺序 |
| --- | --- | --- |
| 多数页面逻辑集中在 `App.vue` 和 `Workbench.vue` | 后续接接口和维护困难 | P4-004 前端页面结构调整时优先处理 |
| 当前大量数据来自 `frontend/src/data` mock 文件 | 不能支撑真实联调 | P4-005 接口契约确认后逐步替换 |
| 部分中文文案疑似存在编码异常 | 课堂演示观感受影响 | 前端结构调整时同步修正文案 |
| 管理员审核、维护者审核、任务发布都混在同一工作台或主文件中 | 角色职责不够清晰 | 拆为独立组件，并按角色入口展示 |
| 成长页没有明确独立页面 | “轻量成长”展示入口不够清楚 | 保留工作台成长记录，必要时增加个人成长入口 |

## 5. P4-003 验收结论

P4-003 的目标是清点现有前端页面，并列出登录、悬赏任务板、任务详情、工作台、提交柜台、管理员审核、成长页的保留、调整或删除项。

本次清点已完成，结论如下：

- 登录/入口页：保留，后续接平台自有账号体系。
- 悬赏任务板：保留，后续接任务列表和筛选接口。
- 任务详情：保留，后续接任务详情、接取和状态流转接口。
- 工作台：保留但需要拆分，优先服务仓库操作、PR、审核反馈和成长记录。
- 提交柜台：保留，后续接成果提交和 PR 校验接口。
- 管理员审核：保留，后续拆为独立页面并接审核接口。
- 成长页：保留轻量成长能力，暂不做复杂成就系统。
- 排行榜与徽章：作为 P2 增强功能，不阻塞 MVP 主链路。
