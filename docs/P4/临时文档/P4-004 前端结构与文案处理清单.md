# P4-004 前端结构与文案处理清单

## 1. 任务边界

P4-004 对应看板任务为“修复中文乱码和术语口径”，直接验收标准是：

- 页面无明显中文乱码。
- 术语统一为：冒险家、委托人、管理员、悬赏任务、接取、提交审核。
- 页面表达与 Git Guild MVP 需求一致。

P4-003 清点出的其他问题也记录在本清单中，但不全部阻塞 P4-004。结构拆分、接口替换、成长页增强、排行榜和徽章等内容应转交到后续任务处理。

## 2. P4-004 必须处理项

| 编号 | 处理项 | 涉及位置 | 处理方式 | 验收标准 |
| --- | --- | --- | --- | --- |

## 4. 建议执行顺序

1. 先修复全局可见乱码，优先处理 `App.vue`、`QuestDetail.vue`、`Workbench.vue`。
2. 再统一角色、任务、动作术语。
3. 然后修复 Git 操作边界、Gitea 同步、管理员审核、轻量成长等关键说明。
4. 最后走一遍演示路径，检查是否还有乱码、术语冲突或职责误导。

## 5. 演示路径检查清单

| 路径 | 检查点 |
| --- | --- |
| 登录入口 -> 悬赏任务板 | 角色、按钮、任务列表、筛选项无乱码，术语统一 |
| 悬赏任务板 -> 任务详情 -> 接取 | 悬赏任务、完成标准、Issue、仓库、接取说明清楚 |
| 任务详情 -> 工作台 | Git 操作边界清楚，能说明创建分支、commit、PR 在工作台完成 |
| 工作台 -> 提交柜台 | 提交柜台只负责成果登记和提交审核，不混淆 Git 操作 |
| 委托人任务发布/审核 | 委托人负责基于 Issue 发布任务、审核成果 |
| 管理员审核页 | 管理员负责上架审核、退回补充、标记异常 |
| 成长信息 | XP、等级、贡献记录和审核反馈表达清楚 |

## 6. 完成判定

完成 P4-004 后，应满足以下条件：

- 核心演示页面没有明显中文乱码。
- 核心业务术语与需求文档一致。
- 用户能区分任务接取、Git 操作、成果提交、维护者审核、管理员上架审核。
- P2 功能不会干扰 P0/P1 主链路演示。
