# P4-021 维护者审核前端

## 任务目标

实现维护者审核前端，支持委托人查看待审核提交、确认 PR 状态、核对完成标准，并录入审核意见。后端 P4-020 已提供提交审核接口，本任务已将审核提交切换到真实 `reviewApi.reviewSubmission()` 调用。

## 拆分结果

| 拆分项 | 文件 | 说明 |
| --- | --- | --- |
| 委托人审核队列 | `frontend/src/components/MaintainerReviewQueue.vue` | 展示待审核提交、状态、提交人、PR 和提交时间，支持选中提交 |
| 审核详情 | `frontend/src/components/MaintainerReviewDetail.vue` | 展示提交摘要、仓库、PR、分支、commit、完成标准逐项检查和成果材料 |
| 反馈操作 | `frontend/src/components/MaintainerReviewActions.vue` | 支持选择 `APPROVED`、`CHANGES_REQUESTED`、`REJECTED`，录入总结和逐项意见，保存草稿或提交真实审核 |
| 委托人工作台入口 | `frontend/src/pages/workbench/MaintainerWorkbenchPage.vue`、`frontend/src/components/Workbench.vue` | 委托人从公会大厅进入的工作台，复用冒险家式任务/仓库流程，并在原视图切换位置提供进入成果审核台的入口 |
| 维护者审核页 | `frontend/src/pages/workbench/MaintainerReviewPage.vue` | 组合队列、详情、操作三栏，作为 `/maintainer/reviews` 路由页面 |
| 演示数据 | `frontend/src/data/maintainerReview.js` | 提供维护者审核统计、队列展示数据、后端 `submissionId` 和审核结论选项 |

## 接口对接说明

当前审核提交调用 `reviewApi.reviewSubmission(submissionId, payload)`，已对接真实接口：

`POST /api/v1/submissions/{submissionId}/reviews`

请求体字段保持为：

```json
{
  "decision": "APPROVED | CHANGES_REQUESTED | REJECTED",
  "summary": "审核总结",
  "items": [
    {
      "checkpoint": "检查项",
      "passed": true,
      "comment": "逐项反馈"
    }
  ]
}
```

## 注意事项

- 委托人登录后与冒险家一样先进入公会大厅；点击大厅中的“工作台”进入 `/maintainer`，再从工作台身份区域的“进入成果审核台”入口进入 `/maintainer/reviews`。
- `Workbench.vue` 中旧的“冒险家视图 / 委托人视图”切换和旧委托人审核模板已移除，避免与独立审核台重复。
- 任务发布草稿与管理员上架审核属于前置链路，可后续单独整理为委托人发布页或入口。
- P4-020 当前只提供创建提交、提交详情、审核提交接口，尚未提供“维护者待审核列表”接口；因此审核队列仍使用本地演示数据，提交动作不再使用 Mock。
- 本地演示数据中的 `submissionId` 需要与后端数据库中真实提交记录对应，否则真实审核提交会返回未找到或权限错误。
