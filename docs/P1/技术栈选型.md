# ADR 001: 技术栈选择（Vue3 + Spring Boot + MySQL）

## 状态
已接受（Accepted）

## 决策
采用 **Vue 3 + Spring Boot 3.5.13 + MySQL 8.0** 技术栈。

### 前端
- **Vue 3**（Composition API）+ **Vite** 构建工具
- **Element Plus** 或 **Ant Design Vue**（后续选定UI库）

### 后端
- **Spring Boot 3.5.13**（单体架构）
- **Spring Data JPA**（数据访问）
- **Spring Validation**（参数校验）
- **Maven** 构建工具

### 数据库
- **MySQL 8.0**（开发阶段暂用 H2 内存数据库过渡）

## 备选方案

| 方案 | 放弃理由 |
|:---|:---|
| **Node.js 全栈** (Express + React) | 团队无 Node.js 后端经验，npm 生态依赖管理对新手不友好 |
| **Spring Cloud 微服务** | 10周时间不足以完成服务拆分、注册中心、配置中心搭建，过度设计 |
| **Python FastAPI** | 虽然开发快，但团队更熟悉 Java 生态，且 Java 企业级规范更适合课程展示 |

## 后果

### 正面
- 团队有 Java 基础，学习曲线平缓
- Spring Boot 生态成熟，文档丰富，AI 辅助生成代码质量高
- 前后端分离架构，可并行开发
- 企业级技术栈，符合"经典企业栈"定位

### 负面
- 前后端语言切换（JS ↔ Java）对单人全栈开发有认知负担
- Spring Boot 启动较慢，热部署需配置
- 后期部署需要 JRE 环境，比 Node.js 复杂

## 验证
成功启动 Spring Boot 3.5.13 + JDK 17，8080 端口可访问。
