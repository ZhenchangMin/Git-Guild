系统边界如下：
1. 平台自己实现账号、登录、角色与权限管理。
2. 平台用户与 Gitea 用户一一绑定。

7. Git 相关能力包括：
   - 仓库浏览
   - 分支浏览与创建
   - 上传/提交变更并生成 commit
   - 发起 PR
   - 合并 PR
   - Issue 浏览与关联
8. 不包括网页代码编辑器。
9. 管理员是有效业务角色，需要参与审核和异常处理。

4. 基于 Issue 发布悬赏任务
5. 浏览、筛选、查看任务详情
6. 接取任务
7. 创建分支
8. 上传/提交变更并生成 commit
9. 发起 PR、查看 PR、合并 PR
10. 提交任务成果
11. 审核任务成果
12. 成长反馈
13. 推荐相关能力：只有材料明确支持且能溯源时才写入
14. 新手引导/流程指引/示例支持
15. 管理员审核任务清晰度/合规性并决定是否上架
16. 常见异常处理：
   - 导入失败
   - 同步失败
   - 任务描述不清被驳回
   - PR/任务关联异常
   - 权限不足
   - 任务被他人领取

- 用户故事
- 用例图 & 详细用例描述

1. 我明确补充的系统边界与系统范围

- 需求应覆盖正常流和关键异常流
- 角色必须区分清楚：冒险家、委托人、管理员
