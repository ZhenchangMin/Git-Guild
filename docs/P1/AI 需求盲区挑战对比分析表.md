# AI需求盲区挑战对比分析表

| 对比维度            | AI 生成的需求                                                                                                                                                                                                 | 人工调研发掘的需求                                                                                                                                                                                                                              | 分析                                                                                                                                                                                                                  |
| ------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| AI 正确覆盖的需求   | 1) 用户系统：用户注册与登录、个人主页<br />2) 代码托管功能：仓库管理、Issue、Pull Request<br />3) 任务悬赏系统：任务发布与接取、任务状态跟踪                                                                  | 1) 用户系统：Github和邮箱等登录方式<br />2) 代码托管功能<br />3) 任务悬赏系统：任务发布与接取、任务信息清晰、审核结果跟踪                                                                                                                       | AI 在“主要流程建模”和“结构化输出”方面有效，适合快速搭建初稿**骨架**。                                                                                                                                       |
| AI 遗漏的需求       | AI遗漏的需求                                                                                                                                                                                                  | 1) 反馈机制细化：24h/72h/1周分层期望。<br />2) 通知策略细化：**关键事件即时** + **每日汇总**的可配置机制。<br />3) 维护者**审核上限**：如每日 <= 5个任务。<br />4) 减少强惩罚机制：连续贡献机制应可选，默认避免“中断清零”。 | 为什么 AI 会遗漏：<br />1) Prompt 前期偏“格式完整”，对细化角色约束不精确。<br />2) AI 更擅长功能面描述，对运营规则、工作负载和行为激励这类“隐性约束”捕捉不足。<br />3) AI未能从多角色角度考虑，导致需求过于笼统。 |
| AI 捏造的需求       | 1) 社区与互动：复杂社区运营，每个任务提供讨论区。<br />2) 以**排行榜**等虚拟激励作为优先项。<br />3) 数据看板                                                                                          | AI捏造的需求                                                                                                                                                                                                                                    | 这些需求并非“完全不存在”，但不属于当前高优先级或必须项。AI 的问题是**时机与优先级捏造**，把后续功能提前。                                                                                                     |
| AI 表述有歧义的需求 | 1) 验收标准里出现“及时反馈”“清晰标准”等不可测词。<br />2) 角色边界不严：谁可审核、谁可修改、何时可见操作入口有时未明确。<br />3) 状态定义不严：任务在待审核/退回/完成的触发条件与异常回滚条件不总是一致。 | 1) 人工版本用户故事补充了 Given-When-Then 的异常路径和可观察结果。<br />2) 用例和用户故事修订中增加了权限不足、导入失败、PR 关联失败、任务被占用等边界条件。                                                                                    | 歧义主要在“不可测试”和“不可执行”：<br />1) 未量化验收标准。<br />2) 未明确角色与状态流转。                                                                                                                        |
